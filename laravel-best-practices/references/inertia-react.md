Key conventions (short):

- Directories: kebab-case (e.g. `common`, `posts`, `profile`).
- Component files: PascalCase and one component per `.tsx` file (e.g. `Button.tsx`).
- Helpers/hooks: camelCase `.ts` files (e.g. `useCurrentUser.ts`).
- Pages: Suffix with `Page` (e.g. `PostsIndexPage.tsx`) and use a default export so Inertia can resolve them.
- Prefer function declarations (`export function Button() { ... }`) over const arrow assignments for components.
- Group custom hooks under `common/hooks` or `modules/<feature>/hooks`.
- Avoid barrel/index files; import components directly using your alias (e.g. `@/common/button/Button`).
- Keep shadcn-generated components under `resources/js/shadcn` and wrap them in `resources/js/common` when exposing a project-friendly API.

Minimal Button.tsx example (typed function declaration):

```tsx
interface ButtonProps {
  type?: 'button' | 'submit';
  className?: string;
  children: React.ReactNode;
}

export function Button({ type = 'button', className, children }: ButtonProps) {
  return (
    <button type={type} className={className}>
      {children}
    </button>
  );
}
```

Page component example (typed props, default export):

```tsx
import { Head } from '@inertiajs/react';

interface Post {
  id: number;
  title: string;
}

interface PostsIndexPageProps {
  posts: Post[];
}

export default function PostsIndexPage({ posts }: PostsIndexPageProps) {
  return (
    <>
      <Head title="Posts" />
      {posts.map(post => (
        <div key={post.id}>{post.title}</div>
      ))}
    </>
  );
}
```

Form submission example (useForm):

```tsx
import { useForm } from '@inertiajs/react';

export default function CreatePostPage() {
  const { data, setData, post, processing, errors } = useForm({ title: '' });

  function handleSubmit(e: React.FormEvent) {
    e.preventDefault();
    post('/posts');
  }

  return (
    <form onSubmit={handleSubmit}>
      <input value={data.title} onChange={e => setData('title', e.target.value)} />
      {errors.title && <p>{errors.title}</p>}
      <button type="submit" disabled={processing}>Save</button>
    </form>
  );
}
```

Notes and migration tips:

- If migrating from a Blade-only app: move design primitives to `common`, feature code to `modules`, and pages to `pages` mirroring the URL structure.
- Avoid using `axios` or `fetch` for form submissions вЂ” use `useForm` or `router` from `@inertiajs/react` so Inertia handles redirects, validation errors, and flash state automatically.
- Type shared data (auth user, flash messages) with a `SharedProps` interface and access it via `usePage<SharedProps>().props` rather than passing it down through every component layer.
- Keep autogenerated shadcn code isolated and wrap it rather than modifying it inline, to keep upgrades simple.
