# PR Review Workflow

## Trigger
A pull request URL, diff, or review request.

## Steps
1. Identify repo / branch / diff source.
2. Load project-specific standards or review skill.
3. Review for correctness, risk, maintainability, and style.
4. Summarize findings by severity.
5. Return a Telegram-friendly summary if needed.

## Output
- overall verdict
- critical issues
- important improvements
- optional patch suggestions
